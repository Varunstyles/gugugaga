The 5-Phase Roadmap

From theoretical foundation to a fully decentralized, mathematically secure architecture.

1

Phase 1: Foundation & Gap Analysis

Developed the Statement of Purpose and built a 13-paper comparative matrix. Proved to supervisors that standard FL lacks security while full HE is computationally unfeasible for edge devices. Gap identified.

Completed
2

Phase 2: Architectural Design

Mapped out the logic for the SecureFedHE framework before writing code. Designed the exact neural network layer split: final layers get encrypted, intermediate layers get protected with mathematical noise (Differential Privacy).

Completed
3

Phase 3: Prototyping & The "Three Rings"

Heavy engineering phase. Built isolated rings to avoid complexity: Vanilla FL baseline (Ring 1), Selective HE + DP (Ring 2), and Decentralization (Ring 3). Successfully debugged DP noise and proved the ~0.20% accuracy drop target.

Completed
4

Phase 4: Evaluation & Benchmarking

Translating raw CSV data from the Phase 3 experiments into publication-ready graphs that prove the system is 3–5x faster than standard methods without sacrificing accuracy. You are here.

Phase 5: Drafting the Manuscript

Future Work

The "Three Rings" Architecture

We built the system in three isolated rings to manage complexity and prove our claims.

1

Ring 1: Vanilla FL

Built a standard, unencrypted system first. Proved that simulated devices could collaboratively train an AI to ~79% accuracy. This provided our baseline.

PyTorch FedAvg Central Server
2

Ring 2: Selective HE + DP

The core novelty. Wrapped the Ring 1 system in cryptography. Fixed mathematical bugs in Differential Privacy noise scaling. Ran experiments at privacy levels (ε = 10, 20, 50).

CKKS Encryption Gaussian DP Selective
3

Ring 3: Decentralization

Replaced the central server with a peer-to-peer ring network, making the system fully decentralized. Nodes aggregate encrypted data homomorphically as it passes around the ring.
